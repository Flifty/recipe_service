# Сервис для подбора кулинарных рецептов

**Сервис для подбора кулинарных рецептов** — это full-stack веб-приложение, предназначенное для поиска, просмотра, добавления и подбора рецептов по имеющимся у пользователя ингредиентам.

Приложение реализовано в формате клиент-серверной системы. Серверная часть отвечает за обработку запросов, бизнес-логику, авторизацию пользователей и взаимодействие с базой данных. Клиентская часть предоставляет удобный пользовательский интерфейс для работы с каталогом рецептов, избранным, отзывами, оценками и профилем пользователя.

Для разработки проекта использовались Java, Spring Boot, PostgreSQL, TypeScript, React и Vite.

### Функционал

- Регистрация и авторизация пользователей.
- JWT-аутентификация и защита персональных действий.
- Просмотр каталога кулинарных рецептов.
- Поиск рецептов по названию.
- Фильтрация рецептов по категории, типу кухни, времени приготовления и количеству порций.
- Сортировка и пагинация списка рецептов.
- Просмотр подробной информации о блюде.
- Отображение состава рецепта с количеством ингредиентов и единицами измерения.
- Добавление собственных рецептов.
- Редактирование и удаление пользовательских рецептов.
- Добавление рецептов в избранное.
- Просмотр списка избранных блюд.
- Добавление, редактирование и удаление комментариев.
- Выставление и обновление оценки рецепта.
- Отображение средней оценки и количества отзывов.
- Редактирование профиля пользователя.
- Изменение имени пользователя, email и пароля.
- Удаление учетной записи.
- История недавно просмотренных рецептов.
- Умный поиск рецептов по имеющимся ингредиентам.
- Отображение недостающих ингредиентов для приготовления блюда.
- Уведомления об успешных действиях и ошибках.

### Умный поиск по ингредиентам

Одной из основных функций приложения является умный поиск рецептов по ингредиентам.

Пользователь выбирает продукты, которые у него есть, после чего система сравнивает их с составом каждого рецепта. Для каждого блюда определяется количество совпавших и недостающих ингредиентов. Результаты сортируются таким образом, чтобы выше отображались рецепты, для которых не хватает меньшего количества продуктов.

Таким образом, пользователь видит не только блюда, которые можно приготовить полностью, но и близкие варианты с указанием недостающих ингредиентов.

### Запуск приложения через Docker

#### Предварительные требования

Перед началом работы убедитесь, что на вашей системе установлены:

1. Docker;
2. Docker Compose;
3. Git.

#### Клонирование репозитория

git clone https://github.com/Flifty/recipe_service.git

cd recipe_service

#### Запуск приложения

Для запуска backend, frontend и базы данных выполните команду:

docker compose up --build

После завершения запуска приложение будет доступно в браузере.

Клиентская часть:

http://localhost:5173

Серверная часть:

http://localhost:8189

### Локальный запуск без Docker

#### Предварительные требования

Перед началом работы убедитесь, что на вашей системе установлены:

1. Java 17 или выше;
2. Maven;
3. PostgreSQL;
4. Node.js версии 18.x или выше;
5. npm;
6. браузер, поддерживающий JavaScript.

#### Настройка базы данных

Создайте базу данных PostgreSQL:

CREATE DATABASE dishes_db;

Для подключения к базе данных используются переменные окружения:

DB_URL=jdbc:postgresql://localhost:5432/dishes_db

DB_USERNAME=postgres

DB_PASSWORD=postgres

JWT_SECRET=change-this-secret

#### Запуск серверной части

Перейдите в директорию backend-приложения:

cd demo

Запустите приложение:

mvn spring-boot:run

Серверная часть будет доступна по адресу:

http://localhost:8189

#### Запуск клиентской части

Откройте новый терминал и перейдите в директорию frontend-приложения:

cd frontend

Установите зависимости:

npm install

Запустите приложение:

npm run dev

Клиентская часть будет доступна по адресу:

http://localhost:5173
